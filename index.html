Self-described "operating system"

Mirus is a new operating system designed on principles created by Plan 9 from Bell Labs. Mirus is not a fork or clone of Plan 9, but rather an entirely different operating system that extends the functionality of Plan 9.

Mirus is designed primarily for higher-performance machines, such as workstations and gaming rigs, and to be used by those of higher technical knowledge. We also are trying to implement cluster-based computing and advanced load bearing capabilities.

A key motivation in the creation of Mirus was to bring many of the powerful concepts created in Plan 9 to a more general audience, and to build upon what was started there.

Mirus was designed to be elegant and powerful, allowing for tasks that may seem menial on other systems to become easy and fluid.

Features

  • Powerful, Plan 9 based architecture
  • Farily POSIX compliant
  • Dynamic, node based filesystem
  • Designed for modern hardware
  • Forward thinking design
  • Resiliant driver framework
  • Fast and lean microkernel

Download

We do not have bleeding-edge ISO images available currently. In order to test drive Mirus, you will need to build it from the source - which is a relatively painless task. You will want to clone and build the git tree from the master branch. Full build instructions are found in the build instructions file.

Grab the source and perform a simple build with the following commands: git clone https://github.com/joshbeitler/mirus.git mirus
cd mirus
git submodule init
git submodule update
rake The resulting ISO image will be contained in the build directory, and is bootable. You can run it in your favorite virtual machines, burn it to a stick or disk, or run over a network.

Requirements

We recommend that you meet the following system requirements for a good experience:

  • CPU: 64 bit, i686 compatable
  • RAM: 512m+
  • Keyboard

Documentation

You can find documentation for Mirus in the documentation/ directory of the source tree as well as on the wiki. A getting started guide is avalible, as well as various other helpful documents.

Contribute

Mirus is open source and community driven. Any and all are welcome to contribute pieces of source code to be included in the project. Ideas, art, and publicity are all welcome. See our contributing guidelines and contributing FAQ for more information on how you can help shape the future of Mirus.

Get Support

If you run into issues (which we almost guarantee you will), please post them on the issue tracker Please tag and describe your issue in detail, and if any error codes are provided please include them.

We are active on Twitter at @mirusproject, as well as the IRC channel #mirus on irc.freenode.net. The wiki is an excellent place to look if you need a quick reference of Mirus.
